Google Ads API — Application Design Document

Company: Pivotree Inc.  |  Contact: floyd.blaikie@pivotree.com  |  Date: June 2026

1. Application Overview

This document describes the Pivotree Marketing Data Pipeline, an internal Python script that consolidates marketing performance data from multiple platforms into a single private Google Sheet for use by Pivotree's marketing and sales leadership team.

The tool is not a commercial product and will not be distributed to third parties. It is used exclusively by Pivotree employees to monitor marketing KPIs across platforms including Google Ads, HubSpot, LinkedIn Ads, and Google Analytics 4.

2. Google Ads API Usage

Purpose

The Google Ads API is used solely to retrieve aggregate campaign performance metrics for internal reporting. No data is modified, no campaigns are created or edited, and no bid adjustments or automated actions are made through the API.

Data Accessed

ResourceFields RetrievedReason
campaigns name, status, impressions, clicks, cost, conversions, conversion_value Weekly campaign performance summary
ad_groups name, campaign, impressions, clicks, cost Ad group breakdown within campaigns
keywords text, match_type, impressions, clicks, cost, quality_score Keyword performance for search campaigns

API Call Volume

The pipeline is run once per day (or on-demand by a team member). Each run makes approximately 3–5 API calls to retrieve the above data for the current reporting period. Total monthly API call volume is expected to remain under 200 calls.

Authentication

Authentication uses OAuth 2.0 with a refresh token stored in a local environment file on the operator's machine. Credentials are not stored in version control or shared externally.

3. Data Flow

  1. The Python script authenticates to the Google Ads API using OAuth 2.0.
  2. Aggregate metrics are fetched for the configured date range (default: year-to-date).
  3. Data is written directly to a private Google Sheet accessible only to Pivotree employees.
  4. No data is stored in a database, transmitted to third parties, or published publicly.

5. Access Scope & Permissions

The application requests the following OAuth scope:

  • https://www.googleapis.com/auth/adwords — read-only access to campaign data

No write operations, campaign modifications, or billing data access are required or requested.

6. Data Privacy & Security

  • All data retrieved from the Google Ads API remains within Pivotree's internal systems.
  • The Google Sheet output is a private document shared only with named Pivotree employees.
  • OAuth credentials are stored locally and rotated periodically.
  • This tool is not a SaaS product, does not have a public-facing interface, and will never be offered to customers or third parties.
